# 🧠 07 — Conceitos Explicados (com Analogias)

![PASTA 07 Conceitos](https://img.shields.io/badge/PASTA-07%20Conceitos-00A4EF?style=for-the-badge) ![AUDIÊNCIA Não-dev primeiro](https://img.shields.io/badge/AUDIÊNCIA-Não-dev%20primeiro-1A1A1A?style=for-the-badge) ![LEIA Antes do dia 2](https://img.shields.io/badge/LEIA-Antes%20do%20dia%202-737373?style=for-the-badge)


> 🗺 **Você está aqui:** [Kit PT-BR](../README.md) → **Conceitos**

> **Para quem é isto?** Para qualquer pessoa do time — especialmente quem **não programa** — que vai cruzar com siglas como EARS, ADR, Spec-Kit, Agent, bounded context.
>
> **O que você terá ao final de cada arquivo desta pasta:**
>
> 1. Entendimento do conceito **sem** ter feito faculdade de computação
> 2. Uma analogia que gruda na memória
> 3. Saber **quando** o conceito aparece no dia
> 4. Saber **para quem** ele importa mais (sua persona)

> 🍄 **Esta pasta é seu Reino Cogumelo conceitual.** Cada arquivo pega um conceito técnico do workshop e traduz para uma analogia do universo **Super Mario** (ou cozinha, ou ata de reunião). A ideia é fazer o termo grudar antes de você precisar dele.

## ✅ Como saber que você "pegou" os conceitos

Antes de abrir o `01-arqueologia/GUIDE.md` para começar o dia, faça este auto-teste mental:

- [ ] Sei explicar o que é **Spec-Kit** em uma frase (sem ler nada)
- [ ] Sei a diferença entre **persona-kit** (`05-personas/`) e **agent-kit** (`06-agentes-de-estagio/`)
- [ ] Sei quando usar **Ask** vs **Plan** vs **Agent** do Copilot
- [ ] Sei o que é uma **EARS** (não precisa lembrar dos 6 padrões, só do "para que serve")
- [ ] Sei o que é um **ADR** (basta saber: "ata de reunião curta sobre decisão técnica")

Se você marcou 4 de 5 acima, está pronto. Vá para o seu `PERSONA.md` em [`../05-personas/`](../05-personas/).
